Exercise 3

1. An articulated statement and description of the assignment based on the nature of the requirements.
This exercise required working with custom functions and translation functions. The assignment focused on creating animated objects with physics-based movement and rotation.

The exercise demonstrates how custom functions can encapsulate behavior (bouncing, spinning) and how translation functions can position and rotate objects. The bouncing balls create dynamic movement while the spinning baskets add rotational motion, creating a lively animated scene.

Excercise 4

1. An articulated statement and description of the assignment based on the nature of the requirements.
This exercise required converting creature functions into a custom QNMonster class with interactive controls. The assignment focused on object-oriented programming principles, creating interactive game-like elements, and implementing user controls for real-time manipulation of objects.

2. Your explanation on the assignment, in terms of the creative act and technical challenges you were asked to engage in
In this sketch, I converted the creature functions into a custom QNMonster class with interactive controls for "Fruit vs Fruit Monster 2.0". The exercise demonstrates object-oriented programming with classes, arrays, vector movement, and interactive controls. The QNMonster class includes parent and child display methods, transform controls, and multiple action methods for user interaction.

The technical challenges included designing a class structure that encapsulated all monster properties (position, velocity, appearance, behavior) while maintaining the visual style from Exercise 3. Implementing interactive controls (click to change color, arrow keys to resize, keyboard shortcuts to create new objects) required careful event handling and state management. Creating smooth vector movement with boundary collision detection was essential for the monsters to move naturally around the canvas. The color-changing feature that uses fruit color palettes required mapping fruit RGB values to monster body and head colors while maintaining visual coherence.

3. Your Point of view on what was successful and what could be done to improve on it.
Successfully converting the procedural monster functions into a reusable class-based system was a major achievement. The interactive controls work well - clicking monsters to change their colors using fruit palettes, using arrow keys to resize all monsters, and keyboard shortcuts to create new monsters and balloons. The vector movement and boundary collision create natural, organic motion. To improve, I could add more interactive features, create different monster types with varied behaviors, add animation sequences, or implement more complex interactions between monsters and fruit balls.

4. Any additional text observations and additional ideas that help with making your artwork more understandable.
The exercise demonstrates how object-oriented programming can transform procedural code into more maintainable and extensible systems. The QNMonster class encapsulates all monster behavior, making it easy to create multiple instances with varied properties. The interactive controls (click monsters to change color, ↑/↓ arrow keys to increase/decrease size, R to reset, M to create new monster, B to create new balloon) make the artwork engaging and responsive. The fruit color palette integration creates visual coherence between the fruit balls and the monsters, reinforcing the "Fruit vs Fruit Monster" theme. The class structure allows for easy extension - new monster types, behaviors, or interactions could be added by creating subclasses or adding new methods.

Project 2

1. An articulated statement and description of the assignment based on the nature of the requirements.
This project required creating an animated visual narrative using food as a medium. The assignment focused on implementing multi-act animations with particle systems, sine wave movements, and smooth transitions between different visual states.

2. Your explanation on the assignment, in terms of the creative act and technical challenges you were asked to engage in
My Project 2 explores the concept of animated condiments dancing across a hotdog canvas, creating a visual narrative through three distinct acts. The idea was inspired by the playful nature of food art and the dynamic movement of sauces being applied to a hotdog. I wanted to create a three-act animation where condiments (ketchup, mayonnaise, and radish) would move across the screen in different patterns - first horizontally as they're being applied, then with smoke effects, and finally vertically as they dance across the full screen height.

Several problems were encountered during development. Initially, the frame rate was set to 700 fps, which was too fast for recording and made the animation difficult to capture properly. Getting the three acts to transition smoothly at the right moments was challenging, as each act needed exactly 300 frames. The particle system positioning required careful adjustment to ensure smoke particles appeared at the correct height and drifted properly. Converting horizontal sine wave movement to vertical movement for Act 3 was particularly tricky, as was balancing opacity levels for overlapping elements without losing visual clarity.

To solve these issues, I optimized the frame rate to 30 fps for smooth recording while maintaining fluid motion. I implemented a modular act system using frameCount % 900 to create seamless 15-second loops with 5-second acts. The particle systems were refined with separate systems for each smoke source and proper wind forces. Different sine wave frequencies were implemented for each condiment to create varied, organic movement patterns. A gradual black fade-in was added for Act 3 to create smooth visual transitions, and rainbow smoke effects were implemented using trigonometric color mapping.

3. Your Point of view on what was successful and what could be done to improve on it.
Successfully creating a three-act animation with smooth transitions was a major achievement. The particle systems, sine wave movements, and rainbow smoke effects all worked well together. The modular act system created seamless loops. To improve, I could add more condiments, create better types of movements, add interactive controls, or extend the animation to include more acts or narrative elements.

4. Any additional text observations and additional ideas that help with making your artwork more understandable.
The final result is a whimsical animation that transforms a simple hotdog into a canvas for dynamic, colorful condiment choreography, demonstrating how food can become a medium for abstract artistic expression. The three-act structure creates a narrative arc: application (Act 1), transformation with smoke (Act 2), and dance/celebration (Act 3). The use of different sine wave frequencies for each condiment creates organic, varied movement patterns that feel natural despite being mathematically generated. The rainbow smoke effects add visual interest and demonstrate advanced color manipulation techniques.
