1. An articulated statement and description of the assignment based on the nature of the requirements.
This exercise required creating an interactive sound-based game using p5.js and p5.sound. The assignment focused on implementing audio playback, user interaction, scoring systems, and scene management to create a playful media interface.

2. Your explanation on the assignment, in terms of the creative act and technical challenges you were asked to engage in
Sound Matching Game — A playful ear-training mini-lab where instrument images map to real sounds; clicks trigger audio, scoring, and a bonus stage. It explores the "playful media interface" context: simple visuals and immediate feedback guide you to learn by listening, while animated sprites keep the space lively. The technical challenges included implementing SceneManager for multiple scenes (title/instructions/game/bonus), synchronizing audio playback with visual feedback, creating a scoring system, and ensuring smooth transitions between game states.

Libraries used: p5.js (p5.min.js - canvas drawing, images, text), p5.sound (p5.sound.min.js - loadSound() and sound.play() for instrument sounds), SceneManager (scenemanager.js - SceneManager for Title/Instructions/Game/Score/Bonus scene transitions), and p5.play (p5.play.js - Sprite class for animated background sprites). The SceneManager library was crucial for organizing multiple game scenes and managing state transitions. The p5.sound library enabled audio playback and synchronization with visual interactions.

3. Your Point of view on what was successful and what could be done to improve on it.
Successfully creating an interactive sound matching game with multiple scenes was a major achievement. The audio-visual connection worked well, and the scoring system provided good feedback. To improve, I could add more instruments, create different difficulty levels, add more visual effects, or implement a high score system.

4. Any additional text observations and additional ideas that help with making your artwork more understandable.
The game demonstrates how sound can be used as a primary interaction mechanism in digital interfaces. The simple visuals and immediate feedback create an accessible learning experience. The animated sprites add visual interest without overwhelming the audio-focused gameplay. The bonus stage provides additional engagement and rewards player success. Best experienced in Chrome/Edge with sound enabled.

Approach to what Exercise 5 taught me relative to Project 4: Exercise 5 was foundational for Project 4 in several key ways. First, it introduced me to SceneManager, which became essential for Project 4's multi-scene interface (intro/help/main/rejection scenes). The experience managing scene transitions and state in Exercise 5 directly informed how I structured Project 4's complex navigation system. Second, Exercise 5 taught me how to use p5.sound for audio feedback (entrance/click/error sounds), which I applied in Project 4 to reinforce mood shifts and user interactions. Third, the exercise demonstrated how to create interactive interfaces with immediate feedback, which helped me design Project 4's tabbed dashboard and dynamic content redrawing. Finally, Exercise 5's approach to using audio cues to guide user experience informed how I used sound in Project 4 to create emotional resonance and highlight the satirical critique of bureaucratic interfaces.

JavaScript Files:
sketch.js (main game logic)
p5.min.js (core p5.js - canvas drawing, images, text)
p5.sound.min.js (audio - loadSound() and sound.play() for instrument sounds)
scenemanager.js (scene management - SceneManager for Title/Instructions/Game/Score/Bonus transitions)
p5.play.js (game library - Sprite class for animated background sprites)

Project 4

Run Project 4

1. An articulated statement and description of the assignment based on the nature of the requirements.
This project required creating an interactive interface that critiques institutional systems through design. The assignment focused on implementing a multi-scene interface with tabbed navigation, audio feedback, and interactive elements that expose systemic barriers, specifically exploring the "institutional/bureaucratic interface" context.

2. Your explanation on the assignment, in terms of the creative act and technical challenges you were asked to engage in
SJSU Student Portal Interface — A satirical mirror of the student center meant to highlight how bureaucracy feels for international students: slow, conditional, and often exclusionary. The "helpful" UI borrows real portal visuals but exposes contradictions—scholarships that exclude you, holds you can't clear, jobs you can't get. This aligns to the "institutional/bureaucratic interface" context: showing how interface tone and sequencing can gate resources.

Built in p5.js with SceneManager, it uses scene transitions for intro/help plus a tabbed main dashboard (To Do, Finances, Holds, Messages, Payment, Enrollment). Tabs redraw content dynamically; audio cues (entrance/click/error) reinforce mood shifts. Enrollment simulates waiting before a button appears; Finances links into a scholarship/job flow that dead-ends; Hold interactions route to rejection scenes to make the blockers explicit. Technical challenges included implementing SceneManager for multiple scenes, creating dynamic tab-based content redrawing, synchronizing audio cues with user interactions, and using timers to simulate bureaucratic delays. Other struggles I encountered included aligning frame boxes and UI elements consistently across different scenes, managing the complexity of multiple layers of interaction challenges, and ensuring that the satirical message came through clearly while maintaining functional interface elements.

3. Your Point of view on what was successful and what could be done to improve on it.
Successfully creating a satirical critique through interface design was a major achievement. The dead-end flows effectively communicate frustration, and the audio cues reinforce the emotional experience. The enrollment timer successfully simulates bureaucratic friction. To improve, I could have more aligned frame boxes and more layers to the challenges, add more instruments and sounds to make it more challenging and engaging, add more interactive elements, create additional rejection scenarios, add more detailed scholarship/job listings, or implement more complex state management for a more realistic simulation.

4. Any additional text observations and additional ideas that help with making your artwork more understandable.
The project demonstrates how interface design can be used as a form of social commentary. By mirroring real portal aesthetics while exposing systemic barriers, the work makes invisible frustrations visible. The dead-end scholarship flows, conditional holds, and delayed enrollment actions all serve to highlight how interface design can gate resources. Audio cues (entrance/click/error) create an emotional arc that reinforces the satirical message. The work shows how "helpful" interfaces can actually be exclusionary when underlying systems create barriers. Best experienced in Chrome/Edge with audio allowed.

JavaScript Files:
sketch.js (main portal logic)
p5.min.js (core p5.js - canvas drawing, UI elements, images, text, rectangles)
p5.sound.min.js (audio - loadSound() and sound.play() for entrance/click/error sound feedback)
scenemanager.js (scene management - SceneManager for Intro/Help/Main/Scholarships/Rejected scene transitions)
